Problem statement
Build a hackathon-ready Somnia Autopilot application that demonstrates chain-native automation and infrastructure primitives (scheduler, alerting, and cross-contract orchestration) with a polished Vite + React + Tailwind dashboard and analytics-oriented execution visibility.
Current state
The repository is empty and has no existing frontend, smart contracts, scripts, or deployment configuration. The target stack needs to include Solidity contracts for reactive automation logic, TypeScript scripts using Somnia Reactivity SDK for subscription setup, and a frontend for job/rule/workflow management plus real-time analytics-style status views.
Proposed changes
Create a monorepo-style structure with separate contracts and app workspaces. In contracts, implement core contracts: an automation registry for jobs/rules, a reactive event handler for system/event-triggered execution, and an orchestrator to run predefined multi-step workflows with execution logs and status transitions. Add Hardhat configuration, deploy scripts, and example interaction scripts that configure Somnia-compatible parameters and emit events for demo validation. In a shared config layer, define network/environment wiring used by both scripts and frontend.
In app, scaffold Vite + React + Tailwind UI with pages/sections for scheduler jobs, alert rules, workflow runs, and analytics summaries (run counts, success/failure ratios, last execution times, active jobs). Integrate wallet connection and contract read/write utilities, plus live refresh loops for recent events and workflow state updates. Add reusable UI components (stat cards, run timeline, status badges, filterable tables, creation drawers/forms) to satisfy the “fancy UI/analytics” requirement while keeping implementation focused on one polished end-to-end scenario.
Wire end-to-end demo flow: create job/rule via UI or script, trigger source event, process reactive handler logic, execute orchestrator steps, and render resulting state/history in UI. Add concise documentation in project README covering setup, environment variables, deployment, running app, and demo script order.
Validate by running build/lint/test commands that exist in each workspace and fixing blocking issues so the project is runnable in a hackathon setting.
